**UP migration (000002_risk_management_schema.up.sql):**

Create 3 tables using DECIMAL for all financial values (from RESEARCH.md: CRITICAL - no float64):

1. **margin_state** table (per-account real-time margin):
   - account_id BIGINT PRIMARY KEY REFERENCES accounts(id) ON DELETE CASCADE
   - equity DECIMAL(20,8) NOT NULL CHECK (equity >= 0)
   - used_margin DECIMAL(20,8) NOT NULL DEFAULT 0
   - free_margin DECIMAL(20,8) NOT NULL COMPUTED AS (equity - used_margin)
   - margin_level DECIMAL(10,2) NOT NULL DEFAULT 0 CHECK (margin_level >= 0)
   - margin_call_triggered BOOLEAN DEFAULT FALSE
   - stop_out_triggered BOOLEAN DEFAULT FALSE
   - last_updated TIMESTAMPTZ NOT NULL DEFAULT NOW()

2. **risk_limits** table (per-account or group limits):
   - id BIGSERIAL PRIMARY KEY
   - account_id BIGINT REFERENCES accounts(id) ON DELETE CASCADE (NULL = group limit)
   - account_group VARCHAR(50) (NULL = account-specific)
   - max_leverage DECIMAL(5,2) NOT NULL CHECK (max_leverage > 0) DEFAULT 30.00
   - max_open_positions INT NOT NULL DEFAULT 50
   - max_position_size_lots DECIMAL(10,2)
   - daily_loss_limit DECIMAL(20,8)
   - max_drawdown_pct DECIMAL(5,2)
   - margin_call_level DECIMAL(5,2) DEFAULT 100.00 NOT NULL
   - stop_out_level DECIMAL(5,2) DEFAULT 50.00 NOT NULL
   - CHECK (margin_call_level > stop_out_level)
   - CONSTRAINT unique_account_or_group UNIQUE NULLS NOT DISTINCT (account_id, account_group)

3. **symbol_margin_config** table (ESMA regulatory limits):
   - symbol VARCHAR(20) PRIMARY KEY
   - asset_class VARCHAR(20) NOT NULL CHECK (asset_class IN ('forex_major', 'forex_minor', 'stock', 'crypto', 'commodity', 'index'))
   - max_leverage DECIMAL(5,2) NOT NULL CHECK (max_leverage > 0)
   - margin_percentage DECIMAL(5,4) NOT NULL (computed as 1/max_leverage * 100)
   - contract_size DECIMAL(20,8) NOT NULL DEFAULT 100000
   - tick_size DECIMAL(10,8) NOT NULL DEFAULT 0.0001
   - tick_value DECIMAL(20,8) NOT NULL DEFAULT 10

Create indexes:
- CREATE INDEX idx_margin_state_last_updated ON margin_state(last_updated)
- CREATE INDEX idx_risk_limits_account ON risk_limits(account_id) WHERE account_id IS NOT NULL
- CREATE INDEX idx_risk_limits_group ON risk_limits(account_group) WHERE account_group IS NOT NULL
- CREATE INDEX idx_symbol_margin_config_asset_class ON symbol_margin_config(asset_class)

**DOWN migration (000002_risk_management_schema.down.sql):**
DROP TABLE symbol_margin_config, risk_limits, margin_state CASCADE;

**CRITICAL (from RESEARCH.md):** Use DECIMAL for ALL financial values. Float64 causes precision errors (LSE halt, German bank €12M fine). PostgreSQL DECIMAL/NUMERIC provides exact decimal arithmetic.

```go
import (
	"context"
	"time"
	"github.com/jackc/pgx/v5/pgxpool"
)

// MarginState represents real-time margin state per account
type MarginState struct {
	AccountID           int64
	Equity              string // DECIMAL as string to avoid float precision issues
	UsedMargin          string
	FreeMargin          string
	MarginLevel         string
	MarginCallTriggered bool
	StopOutTriggered    bool
	LastUpdated         time.Time
}

type MarginStateRepository struct {
	pool *pgxpool.Pool
}

func NewMarginStateRepository(pool *pgxpool.Pool) *MarginStateRepository {
	return &MarginStateRepository{pool: pool}
}
```

**Methods to implement:**
1. `GetByAccountID(ctx context.Context, accountID int64) (*MarginState, error)` - SELECT by account_id
2. `Upsert(ctx context.Context, state *MarginState) error` - INSERT ON CONFLICT UPDATE (real-time updates)
3. `GetByAccountIDWithLock(ctx context.Context, tx pgx.Tx, accountID int64) (*MarginState, error)` - SELECT FOR UPDATE (for concurrent safety during margin calculation)

Use parameterized queries ($1, $2) to prevent SQL injection. Return pgx errors directly (don't wrap with custom errors yet).

**CRITICAL:** All DECIMAL columns returned as strings, NOT float64. Conversion to decimal.Decimal happens in business logic layer.
