<task type="auto">
  <name>Task 1: Implement trailing stop price adjustment algorithm</name>
  <files>backend/internal/core/order_monitor.go, backend/internal/core/engine.go</files>
  <action>
    In order_monitor.go:
    - Add UpdateTrailingStops(currentPrices map[string]float64) method
    - For each order with Type=TRAILING_STOP and trailing_delta set:
      * Get current price for symbol (bid for sells, ask for buys)
      * Calculate new trigger based on best price:
        - Buy (long position): new_trigger = current_ask - trailing_delta
        - Sell (short position): new_trigger = current_bid + trailing_delta
      * Only update if new trigger is MORE favorable than current trigger:
        - Buy: new_trigger > old_trigger (stop moves up with rising price)
        - Sell: new_trigger < old_trigger (stop moves down with falling price)
      * If updated: orderRepo.Update() to persist new trigger_price
    - Call UpdateTrailingStops() before CheckPriceTriggers() in monitor loop

    In engine.go:
    - Trailing stop execution uses same ExecuteTriggeredOrder() from Plan 01
    - No changes needed (already handles any order with trigger_price)

    Algorithm example (Buy position, trailing_delta=0.0020):
    - Initial: trigger=1.0950 (entry 1.0970 - 0.0020)
    - Price moves to 1.0990 → trigger updates to 1.0970 (1.0990 - 0.0020)
    - Price moves to 1.1010 → trigger updates to 1.0990 (1.1010 - 0.0020)
    - Price reverses to 1.0985 → trigger stays 1.0990, position closes

    Use repository pattern: orderRepo.Update(order) to persist trigger changes.
    Add RWMutex locking when updating triggers (prevent race conditions).
  </action>

    - go test ./backend/internal/core/... passes
    - Manual test: Set trailing stop, move price favorable → trigger follows
    - Manual test: Price reverses → position closes at adjusted stop level
  </verify>
  <done>

  </done>
