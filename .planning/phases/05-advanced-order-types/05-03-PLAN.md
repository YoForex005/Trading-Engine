<task type="auto">
  <name>Task 1: Implement pending order trigger and execution logic</name>
  <files>backend/internal/core/order_monitor.go, backend/internal/core/engine.go</files>
  <action>
    In order_monitor.go:
    - Add CheckPendingOrders(currentPrices map[string]float64) method
    - For each order with Type=BUY_LIMIT|SELL_LIMIT|BUY_STOP|SELL_STOP and Status=PENDING:
      * Get current bid/ask for symbol
      * Check trigger conditions:
        - BUY_LIMIT: triggers when ask <= trigger_price (price drops to limit)
        - SELL_LIMIT: triggers when bid >= trigger_price (price rises to limit)
        - BUY_STOP: triggers when ask >= trigger_price (breakout up)
        - SELL_STOP: triggers when bid <= trigger_price (breakout down)
      * If triggered: call engine.ExecutePendingOrder(orderID, currentPrice)
    - Call CheckPendingOrders() in monitor loop (after trailing stop updates)

    In engine.go:
    - ExecutePendingOrder(orderID int64, fillPrice float64) method
    - Load order from orderRepo
    - Validate: order is PENDING, price is still valid
    - Create Position:
      * Symbol=order.Symbol
      * Side=order.Side (BUY/SELL)
      * Volume=order.Volume
      * OpenPrice=fillPrice (actual fill price, may differ slightly from trigger)
      * Set order.SL/TP on position if specified
    - Update order: Status=FILLED, FilledPrice=fillPrice, FilledAt=now
    - Save position via positionRepo.Create()
    - Save order via orderRepo.Update()
    - Broadcast position/account update via WebSocket

    Follow existing ExecuteMarketOrder() pattern from engine.go.
    Use repository pattern for all database operations.
    Add RWMutex locking for thread safety.
  </action>
  <verify>
    - go build ./backend/... succeeds
    - go test ./backend/internal/core/... passes
    - Manual test: Place Buy Limit below market → price drops → position opens
    - Manual test: Place Sell Stop below market → price drops → position opens
  </verify>
  <done>
    - Pending orders trigger at correct price levels
    - Positions open automatically with correct parameters
    - Order status updates to FILLED
    - Changes persist to database
  </done>

      * SL, TP if provided

    Add GET /api/orders endpoint to list pending orders for account:
