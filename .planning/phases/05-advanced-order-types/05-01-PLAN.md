<task type="auto">
  <name>Task 2: Implement SL/TP monitoring and execution engine</name>
  <files>backend/internal/core/order_monitor.go, backend/internal/core/engine.go</files>
  <action>
    Create backend/internal/core/order_monitor.go:
    - OrderMonitor struct with ticker (checks every 100ms)
    - CheckPriceTriggers(currentPrices map[string]float64) method
    - For each order with Status=PENDING and trigger_price set:
      * Compare current price vs trigger_price based on order side
      * SL Buy: triggers when bid <= trigger_price
      * SL Sell: triggers when ask >= trigger_price
      * TP Buy: triggers when bid >= trigger_price
      * TP Sell: triggers when ask <= trigger_price
    - Call engine.ExecuteTriggeredOrder(orderID) when conditions met

    In engine.go:
    - ExecuteTriggeredOrder(orderID int64) method
    - Load order from repository
    - If parent_position_id set: close that position at current price
    - Mark order as FILLED, update filled_price and filled_at
    - Persist to database via repository
    - Broadcast position/account update via WebSocket

    Start monitor goroutine in main.go (after engine initialization).

    Use Phase 2 repository pattern: inject orderRepo via constructor.
    Follow existing RWMutex patterns for thread safety.
  </action>
  <verify>
    - go test ./backend/internal/core/... passes (create basic test for trigger logic)
    - Monitor goroutine starts without panic
    - Manual test: Set SL on position, move price past trigger, verify position closes
  </verify>
  <done>
    - OrderMonitor service running in background
    - SL/TP orders execute automatically when price hits trigger
    - Position closes with correct P/L calculation
    - Changes persist to database
  </done>
</task>
