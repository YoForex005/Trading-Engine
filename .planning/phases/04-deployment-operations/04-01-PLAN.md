  <files>backend/Dockerfile</files>
  <action>
Create multi-stage Dockerfile for Go backend following research best practices:

**Build stage (golang:1.21-alpine)**:
1. Set WORKDIR /app
2. Copy go.mod and go.sum FIRST (for layer caching)
3. RUN go mod download
4. Copy all source code
5. Build with: CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -trimpath -ldflags="-s -w" -o server ./cmd/server
   - -trimpath removes file system paths
   - -ldflags="-s -w" strips debug info and symbol table

**Runtime stage (gcr.io/distroless/static:nonroot)**:
1. WORKDIR /app
2. COPY --from=builder /app/server .
3. USER nonroot:nonroot (security - never run as root)
4. EXPOSE 8080
5. CMD ["./server"]

Why distroless: No shell, no package manager, minimal attack surface (2-5MB vs 800MB with alpine+shell).
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Create frontend multi-stage Dockerfile with nginx</name>
  <files>clients/desktop/Dockerfile, clients/desktop/nginx.conf</files>
  <action>
Create multi-stage Dockerfile for React+Vite frontend:

**Build stage (node:20-alpine)**:
1. WORKDIR /app
2. Copy package.json and bun.lock FIRST (layer caching)
3. RUN npm install -g bun && bun install
4. Copy all source code
5. ENV NODE_ENV=production
6. RUN bun run build (outputs to dist/)

**Runtime stage (nginx:1.25-alpine)**:
1. Remove default nginx config: RUN rm /etc/nginx/conf.d/default.conf
2. COPY custom nginx.conf /etc/nginx/conf.d/
3. COPY --from=builder /app/dist /usr/share/nginx/html
4. EXPOSE 80
5. CMD ["nginx", "-g", "daemon off;"]

Create nginx.conf with:
- SPA routing: try_files $uri $uri/ /index.html (all routes serve index.html)
- Gzip compression: gzip on, types text/plain text/css application/json application/javascript
- MIME types: application/javascript for .js and .mjs (Vite ES modules)
- Cache static assets: location ~* \.(js|css|png|jpg|svg)$ with Cache-Control headers

Why nginx: Battle-tested static file serving, 20-30MB final image.
  </action>
