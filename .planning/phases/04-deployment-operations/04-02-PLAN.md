<task type="auto">
  <name>Task 1: Create docker-compose.yml with all services</name>
  <files>docker-compose.yml</files>
  <action>
Create docker-compose.yml (version 3.8) with 6 services:

**1. backend**:
- build: context: ./backend, target: (use build stage for hot reload in dev)
- ports: "8080:8080"
- environment: DATABASE_URL, REDIS_URL (reference db and redis services)
- volumes: ./backend:/app (mount source for hot reload)
- depends_on: [db, redis]

**2. frontend**:
- build: context: ./clients/desktop, target: (use build stage for dev server)
- ports: "5173:5173" (Vite dev server)
- volumes: ./clients/desktop:/app, /app/node_modules (don't override node_modules)
- environment: VITE_API_URL=http://localhost:8080

**3. db** (postgres:16-alpine):
- image: postgres:16-alpine
- environment: POSTGRES_DB=trading_engine, POSTGRES_USER=postgres, POSTGRES_PASSWORD=password
- ports: "5432:5432"
- volumes: postgres_data:/var/lib/postgresql/data

**4. redis** (redis:7-alpine):
- image: redis:7-alpine
- ports: "6379:6379"
- volumes: redis_data:/data

**5. prometheus** (prom/prometheus:latest):
- image: prom/prometheus:latest
- ports: "9090:9090"
- volumes: ./monitoring/prometheus.yml:/etc/prometheus/prometheus.yml, prometheus_data:/prometheus

**6. grafana** (grafana/grafana:latest):
- image: grafana/grafana:latest
- ports: "3000:3000"
- environment: GF_SECURITY_ADMIN_PASSWORD=admin
- volumes: grafana_data:/var/lib/grafana, ./monitoring/grafana/dashboards:/etc/grafana/provisioning/dashboards
- depends_on: [prometheus]

**Volumes**: postgres_data, redis_data, prometheus_data, grafana_data (persist data across restarts)

**Networks**: Use default bridge network (services reference each other by service name).
  </action>
  <verify>
docker-compose config (validates YAML syntax)
docker-compose up -d (starts all services)
docker-compose ps (shows all 6 services running)
  </verify>
  <done>All services start successfully and can communicate via service names</done>
</task>

<task type="auto">
  <name>Task 2: Create Prometheus scrape configuration</name>
  <files>monitoring/prometheus.yml</files>
  <action>
Create Prometheus configuration to scrape backend metrics:

```yaml
global:
  scrape_interval: 10s  # Scrape every 10 seconds
  evaluation_interval: 10s

scrape_configs:
  - job_name: 'trading-engine-backend'
    static_configs:
      - targets: ['backend:8080']  # Service name from docker-compose
    metrics_path: '/metrics'  # Standard Prometheus endpoint
    scrape_interval: 5s  # More frequent for trading platform
```

This assumes backend will expose /metrics endpoint (implemented in plan 04-04).
  </action>
  <verify>
docker-compose up prometheus (starts Prometheus)
curl localhost:9090/-/healthy (returns 200)
Visit http://localhost:9090/targets (shows backend target, may be down until 04-04 implements /metrics)
  </verify>
  <done>Prometheus configured to scrape backend metrics endpoint</done>
</task>
