Create comprehensive Docker guide covering:

**Building Images:**
```bash
# Backend
cd backend
docker build -t trading-engine-backend:latest .

# Frontend
cd clients/desktop
docker build -t trading-engine-frontend:latest .
```

**Image Details:**
- Backend: Multi-stage build (golang:1.21-alpine → distroless)
- Final size: ~5-10MB
- Frontend: Multi-stage build (node:20-alpine → nginx:1.25-alpine)
- Final size: ~30-50MB

**Running Containers:**
```bash
# Backend
docker run -p 8080:8080 \
  -e DATABASE_URL=postgresql://user:pass@host:5432/trading_engine \
  -e REDIS_URL=redis:6379 \
  trading-engine-backend:latest

# Frontend
docker run -p 80:80 trading-engine-frontend:latest
```

**Registry Publishing:**
```bash
# Tag for registry
docker tag trading-engine-backend:latest ghcr.io/org/trading-engine-backend:v1.0.0

# Push to GitHub Container Registry
docker push ghcr.io/org/trading-engine-backend:v1.0.0
```

**Best Practices:**
- Use specific version tags (not :latest in production)
- Set resource limits (--memory, --cpus)
- Use health checks in docker run
- Run as non-root user (already configured in Dockerfiles)

**Security:**
- Images use distroless base (minimal attack surface)
- No shell access in production images
- Secrets via environment variables, never baked in

  <action>
Create local development environment guide:

**Prerequisites:**
- Docker and Docker Compose installed
- Go 1.21+ (for backend development)
- Bun (for frontend development)

**Quick Start:**
```bash
# Clone repository
git clone https://github.com/org/trading-engine.git
cd trading-engine

# Start all services
docker-compose up -d

# Check status
docker-compose ps

# View logs
docker-compose logs -f backend
```

**Service URLs:**
- Backend API: http://localhost:8080
- Frontend UI: http://localhost:3000
- Prometheus: http://localhost:9090
- Grafana: http://localhost:3001
- PostgreSQL: localhost:5432
- Redis: localhost:6379

**Development Workflow:**

*Backend (Go):*
```bash
cd backend
go mod download
go run cmd/server/main.go
# OR use docker-compose for just dependencies:
docker-compose up -d db redis
```

*Frontend (React):*
```bash
cd clients/desktop
bun install
bun run dev
# Hot reload enabled on http://localhost:5173
```

**Database Management:**
```bash
# Run migrations
docker-compose exec backend ./server migrate up

# Access PostgreSQL
docker-compose exec db psql -U postgres -d trading_engine

# Reset database
docker-compose down -v  # Removes volumes
docker-compose up -d
```

**Troubleshooting:**
- Port conflicts: Check `docker-compose ps` and modify ports in docker-compose.yml
- Database not ready: Wait 5-10s after `docker-compose up` before connecting
- Frontend proxy errors: Ensure backend is running on port 8080
- Redis connection issues: Check `docker-compose logs redis`

**Environment Variables:**
Create `.env` file in project root:
```
DATABASE_URL=postgresql://postgres:password@localhost:5432/trading_engine
REDIS_URL=localhost:6379
LOG_LEVEL=debug
```

Create operations runbook (OPERATIONS.md):

**Health Checks:**
```bash
# Liveness (process alive)
curl http://localhost:8080/health/live

# Readiness (dependencies healthy)
curl http://localhost:8080/health/ready
```

**Common Issues:**

*Issue: High order latency*
- Check: `trading_order_duration_seconds` histogram
- Possible causes: Database slow queries, Redis cache misses
- Solution: Check slow query log, verify Redis connectivity

*Issue: Database connection errors*
- Check: `/health/ready` returns 503
- Logs: `docker-compose logs db`
- Solution: Verify DATABASE_URL, check connection pool settings

*Issue: Redis cache not working*
- Check: Redis logs for connection errors
- Test: `docker-compose exec redis redis-cli PING`
- Solution: Verify REDIS_URL, check network connectivity

*Issue: Frontend 502 Bad Gateway*
- Check: Backend health endpoint
- Possible causes: Backend not running, nginx misconfiguration
- Solution: Restart backend, check nginx.conf proxy settings

**Backup and Recovery:**
```bash
# Manual backup
docker-compose exec backend bash scripts/backup-db.sh

# List backups
ls -lh /backups/postgres/

# Restore from backup
gunzip < /backups/postgres/trading_engine_20260116_120000.sql.gz | \
  docker-compose exec -T db psql -U postgres -d trading_engine
```

**Performance Optimization:**
- LP lookups: O(1) map-based (implemented in 04-06)
- Tick caching: 60s TTL in Redis
- OHLC caching: Variable TTL (1hr-7d)
- Database indexes: Check EXPLAIN ANALYZE for slow queries

**Scaling Considerations:**
- Horizontal: Run multiple backend instances behind load balancer
- Vertical: Increase DATABASE_URL connection pool size
- Caching: Redis cluster for high availability
- Database: PostgreSQL read replicas for analytics queries

**Security Checklist:**
- [ ] Change default passwords in .env
- [ ] Use secrets management (not .env) in production
- [ ] Enable TLS for PostgreSQL connections
- [ ] Restrict Redis access (requirepass in redis.conf)
- [ ] Run containers as non-root (already configured)
- [ ] Keep base images updated (rebuild monthly)

**Monitoring Alerts (recommended):**
- Order latency P95 > 100ms
- Database connection pool exhaustion
- Redis eviction rate > 10/s
- Disk usage > 80%
- Backup failures

Create CI/CD pipeline documentation:

**GitHub Actions Workflows:**

1. **Backend CI** (.github/workflows/backend.yml)
   - Triggers: Push to backend/**, go.mod, go.sum
   - Jobs: Test → Build → Publish
   - Docker caching: BuildKit with type=gha

2. **Frontend CI** (.github/workflows/frontend.yml)
   - Triggers: Push to clients/desktop/**, package.json
   - Jobs: Typecheck → Test → Build → Publish
   - Bun for faster dependency installation

3. **Database Backups** (.github/workflows/backup.yml)
   - Schedule: Every 6 hours (cron: '0 */6 * * *')
   - Artifact retention: 30 days
   - Manual trigger: workflow_dispatch

**Running Workflows:**
```bash
# Trigger manually
gh workflow run backend.yml

# View recent runs
gh run list --workflow=backend.yml

# Watch logs
gh run watch
```

**Path Filtering (Monorepo Optimization):**
- Uses dorny/paths-filter@v3
- Only runs affected workflows
- Example: Backend change doesn't trigger frontend build
- Saves 50-70% CI time

**Docker Image Publishing:**
- Registry: GitHub Container Registry (ghcr.io)
- Tags:
  - `latest` on main branch
  - `sha-{git-sha}` for every commit
  - `v{version}` on release tags
- Authentication: GITHUB_TOKEN (automatic)

**Secrets Configuration:**
Required secrets in repository settings:
- DB_PASSWORD (for backup workflow)
- (Production: Add cloud provider credentials)

**Best Practices:**
- Cache dependencies (go mod cache, bun install cache)
- Run tests in parallel where possible
- Fail fast (stop on first error)
- Use matrix builds for multiple Go/Node versions (if needed)

**Local Testing:**
```bash
# Install act (GitHub Actions local runner)
brew install act

# Run workflow locally
act -j backend-test

# With secrets
act -j backend-test -s DB_PASSWORD=test
```
