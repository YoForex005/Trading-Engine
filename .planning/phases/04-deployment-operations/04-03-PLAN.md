<task type="auto">
  <name>Task 1: Create backend CI workflow with path filtering</name>
  <files>.github/workflows/backend.yml</files>
  <action>
Create GitHub Actions workflow for backend:

**Trigger**: on push and pull_request

**Jobs**:
1. **detect-changes**: Uses dorny/paths-filter@v3 with filters:
   - backend: ['backend/**', 'go.mod', 'go.sum']
   - Outputs: backend: true/false

2. **backend-test**:
   - needs: detect-changes
   - if: needs.detect-changes.outputs.backend == 'true'
   - runs-on: ubuntu-latest
   - Steps: checkout, setup-go@v5 (go-version: '1.21', cache: true), go test -race ./...

3. **backend-build**:
   - needs: backend-test
   - if: github.ref == 'refs/heads/main'
   - Uses docker/build-push-action@v5 with:
     - context: ./backend
     - push: true
     - tags: ghcr.io/${{ github.repository }}/backend:latest
     - cache-from: type=gha
     - cache-to: type=gha,mode=max

Include docker/setup-buildx-action@v3 and docker/login-action@v3 (ghcr.io registry).
  </action>
  <verify>
git add .github/workflows/backend.yml
git commit -m "ci: add backend workflow"
git push (triggers workflow)
GitHub Actions shows workflow running
  </verify>
  <done>Backend workflow runs on push, only when backend/ changes, tests pass</done>
</task>
