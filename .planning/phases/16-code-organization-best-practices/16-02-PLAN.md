# Plan: Structured Logging Migration

**Phase:** 16 - Code Organization & Best Practices
**Wave:** 1 (parallel with 16-01)
**Dependencies:** None
**Scope:** ~50% context budget

---

## Goal

Replace 320 unstructured logging calls (fmt.Printf, fmt.Println, log.Printf) with slog structured logging, making logs queryable and production-ready for monitoring systems.

---

## Must-Haves

### Truths (what must be TRUE when done)

1. All fmt.Printf/fmt.Println/log.Printf calls replaced with slog equivalents
2. slog logger configured with JSON handler for production
3. Critical business events logged with structured fields (account_id, symbol, order_id, etc.)
4. Log levels used appropriately (Info, Warn, Error, Debug)
5. No unstructured logging remains in production code paths

### Artifacts (files that must exist)

```
backend/internal/shared/logger/logger.go  # Centralized logger setup
backend/cmd/server/main.go                # Logger initialization in main
```

### Key Links (code that must connect correctly)

- All packages use shared logger instance (via dependency injection or package-level var)
- HTTP handlers pass logger via context
- Error logs include error field: `logger.Error("msg", "error", err)`

## Tasks

### Task 1: Create Centralized Logger Package

**What:** Set up slog logger with JSON handler in `internal/shared/logger/`

**Why:** Standard library structured logging, zero dependencies, future-proof

**Details:**
```go
// backend/internal/shared/logger/logger.go
package logger

import (
	"log/slog"
	"os"
)

var Default *slog.Logger

// Init initializes the global logger
func Init(level slog.Level) {
	handler := slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
		Level: level,
		// Add source for debugging (file:line)
		AddSource: false,  // Enable in dev if needed
	})

	Default = slog.New(handler)
	slog.SetDefault(Default)
}

// WithContext creates a logger with additional context fields
func WithContext(fields ...any) *slog.Logger {
	return Default.With(fields...)
}
```

**Initialize in main:**
```go
// backend/cmd/server/main.go
import "github.com/trading-engine/internal/shared/logger"

func main() {
	// Determine log level from env
	logLevel := slog.LevelInfo
	if os.Getenv("DEBUG") == "true" {
		logLevel = slog.LevelDebug
	}

	logger.Init(logLevel)
	logger.Default.Info("Trading Engine starting", "version", "1.0.0")

	// ... rest of main
}
```

**Checkpoint:** Logger initializes successfully, outputs JSON to stdout

### Task 2: Migrate High-Priority Logging (bbook/engine.go)

**What:** Replace unstructured logging in core trading engine with structured slog calls

**Why:** engine.go has most critical business logic (position open/close, margin, P&L)

**Details:**

Current patterns to replace:
```go
// BEFORE
fmt.Printf("Position opened: %+v\n", position)
fmt.Println("Closing position", position.ID)
log.Printf("Error closing position: %v", err)
```

After:
```go
// AFTER
import "github.com/trading-engine/internal/shared/logger"

logger.Default.Info("position opened",
	"position_id", position.ID,
	"account_id", position.AccountID,
	"symbol", position.Symbol,
	"volume", position.Volume,
	"open_price", position.OpenPrice,
)

logger.Default.Info("closing position", "position_id", position.ID)

logger.Default.Error("failed to close position",
	"position_id", position.ID,
	"error", err,
)
```

**Focus areas in engine.go:**
- `ExecuteBuy()` / `ExecuteSell()` - position open events
- `ClosePosition()` - position close events
- `CalculateMargin()` - margin calculation logs
- Error paths - all should use logger.Error with "error" field

**Checkpoint:** engine.go has zero fmt.Print*/log.Print* calls

### Task 5: Update Tests to Handle Structured Logging

**What:** Ensure tests don't break due to logger changes

**Why:** Tests may mock or capture log output

**Details:**

**For tests that don't care about logs:**
```go
func TestSomething(t *testing.T) {
	// Initialize logger to discard output
	logger.Init(slog.LevelError)  // Only show errors in tests

	// ... test code ...
}
```

**For tests that verify logging:**
```go
func TestLoggingBehavior(t *testing.T) {
	// Capture log output
	var buf bytes.Buffer
	handler := slog.NewJSONHandler(&buf, &slog.HandlerOptions{
		Level: slog.LevelDebug,
	})
	logger.Default = slog.New(handler)

	// ... trigger code that logs ...

	// Verify log output
	logs := buf.String()
	assert.Contains(t, logs, "position opened")
	assert.Contains(t, logs, "account_id")
}
```

**Checkpoint:** All tests pass (`go test ./...`)

### Task 6: Add Logging Best Practices Documentation

**What:** Document logging guidelines for future development

**Why:** Ensure team uses structured logging correctly

**Details:**

Add to CONTRIBUTING.md or create LOGGING.md:
```markdown
## Logging Guidelines

### Use slog for All Logging

```go
import "github.com/trading-engine/internal/shared/logger"

// Info - normal operations
logger.Default.Info("position opened", "position_id", pos.ID, "symbol", pos.Symbol)

// Warn - unexpected but handled
logger.Default.Warn("retry failed", "attempt", retryCount, "max_retries", maxRetries)

// Error - failures requiring attention
logger.Default.Error("database query failed", "query", sql, "error", err)

// Debug - detailed troubleshooting (disabled in production)
logger.Default.Debug("calculated margin", "margin", margin, "equity", equity)
```

### Always Include Context Fields

- **account_id** - every operation on an account
- **position_id** - position operations
- **order_id** - order operations
- **symbol** - market data or trading operations
- **error** - all error logs MUST include error field

### Log Levels

- **Debug:** Verbose troubleshooting (disabled in production)
- **Info:** Normal operations (position opened, order placed)
- **Warn:** Unexpected but handled (retry after connection loss)
- **Error:** Failures requiring attention (database errors, LP failures)

### DON'T Use fmt.Print* or log.Print*

Unstructured logs are not queryable. Use slog exclusively.
```

## Anti-Patterns to Avoid

- **String concatenation in log messages:** Use fields instead
  ```go
  // BAD
  logger.Info(fmt.Sprintf("Position %s opened", id))

  // GOOD
  logger.Info("position opened", "position_id", id)
  ```

- **Logging sensitive data:** Don't log passwords, API keys, tokens
- **Excessive logging:** Don't log every function call (use Debug level sparingly)
- **Missing error field:** Every logger.Error call MUST include "error" field
