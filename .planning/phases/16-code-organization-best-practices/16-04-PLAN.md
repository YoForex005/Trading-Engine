# Plan: Backend Clean Architecture Refactoring

**Phase:** 16 - Code Organization & Best Practices
**Wave:** 2 (parallel with 16-03)
**Dependencies:** 16-01 (linting), 16-02 (logging)
**Scope:** ~55% context budget

---

## Goal

Refactor backend from flat package structure to clean architecture layers (domain/ports/adapters), splitting large files (913-line engine.go, 854-line api.go) into focused, testable modules with clear separation of concerns.

---

## Must-Haves

### Truths (what must be TRUE when done)

1. Domain logic separated from infrastructure (HTTP, database, WebSocket)
2. Dependencies point inward toward domain (ports define interfaces, adapters implement)
3. No single file exceeds 500 lines (split engine.go and api.go)
4. Business logic testable without HTTP server or database
5. Package structure follows pragmatic clean architecture (internal/domain, internal/ports, internal/adapters)

### Artifacts (files that must exist)

```
backend/internal/
├── domain/              # Pure business logic (entities, value objects)
│   ├── account/
│   │   ├── account.go         # Account entity
│   │   └── account_test.go
│   ├── position/
│   │   ├── position.go        # Position entity with P&L calc
│   │   └── position_test.go
│   ├── order/
│   │   ├── order.go           # Order entity
│   │   └── order_test.go
│   └── trade/
│       ├── trade.go           # Trade entity (immutable)
│       └── trade_test.go
├── ports/               # Interfaces (dependency inversion)
│   ├── repositories.go        # Repository interfaces
│   └── services.go            # Service interfaces
├── adapters/            # Infrastructure implementations
│   ├── http/
│   │   ├── handlers.go        # HTTP handlers (split from api.go)
│   │   ├── middleware.go
│   │   └── routes.go
│   ├── postgres/              # Database repositories (already exists)
│   └── ws/                    # WebSocket adapter
└── shared/              # Already exists (decimal, logger, errors)
```

### Key Links (code that must connect correctly)

- Domain entities have zero dependencies on infrastructure (no database imports in domain/)
- Adapters implement port interfaces
- HTTP handlers depend on ports (interfaces), not concrete repositories
- Tests can mock ports without spinning up database or HTTP server

### Task 1: Extract Domain Entities from engine.go

**What:** Create pure domain entities (Account, Position, Order, Trade) with business logic

**Why:** Separate core business rules from engine orchestration logic

**Details:**

**Account entity:**
```go
// backend/internal/domain/account/account.go
package account

import "github.com/trading-engine/internal/shared/decimal"

// Account represents a trading account
type Account struct {
	ID      string
	Balance decimal.Decimal
	Equity  decimal.Decimal
	// ... other fields
}

// CanOpenPosition checks if account has sufficient funds
func (a *Account) CanOpenPosition(required decimal.Decimal) bool {
	return a.Balance.GreaterThanOrEqual(required)
}

// UpdateBalance adjusts account balance
func (a *Account) UpdateBalance(amount decimal.Decimal) {
	a.Balance = a.Balance.Add(amount)
}

// UpdateEquity recalculates equity based on unrealized P&L
func (a *Account) UpdateEquity(unrealizedPnL decimal.Decimal) {
	a.Equity = a.Balance.Add(unrealizedPnL)
}
```

**Position entity:**
```go
// backend/internal/domain/position/position.go
package position

import "github.com/trading-engine/internal/shared/decimal"

// Position represents an open trading position
type Position struct {
	ID         string
	AccountID  string
	Symbol     string
	Side       string  // "BUY" or "SELL"
	Volume     decimal.Decimal
	OpenPrice  decimal.Decimal
	// ... other fields
}

// CalculatePnL computes unrealized P&L at current price
func (p *Position) CalculatePnL(currentPrice decimal.Decimal) decimal.Decimal {
	priceDiff := currentPrice.Sub(p.OpenPrice)
	if p.Side == "SELL" {
		priceDiff = priceDiff.Neg()
	}
	return priceDiff.Mul(p.Volume)
}

// IsProfitable checks if position has unrealized profit
func (p *Position) IsProfitable(currentPrice decimal.Decimal) bool {
	return p.CalculatePnL(currentPrice).GreaterThan(decimal.Zero)
}
```

**Order and Trade entities:**
- Extract Order struct + validation logic
- Extract Trade struct (immutable, no methods beyond getters)

**Checkpoint:** Domain entities compile, have zero external dependencies (only internal/shared)

### Task 2: Define Port Interfaces

**What:** Create interfaces for repositories and services

**Why:** Dependency inversion - domain defines what it needs, adapters provide it

**Details:**

```go
// backend/internal/ports/repositories.go
package ports

import (
	"context"

	"github.com/trading-engine/internal/domain/account"
	"github.com/trading-engine/internal/domain/position"
	"github.com/trading-engine/internal/domain/order"
	"github.com/trading-engine/internal/domain/trade"
)

// AccountRepository defines account persistence operations
type AccountRepository interface {
	Get(ctx context.Context, id string) (*account.Account, error)
	Update(ctx context.Context, acc *account.Account) error
	List(ctx context.Context) ([]*account.Account, error)
}

// PositionRepository defines position persistence operations
type PositionRepository interface {
	Get(ctx context.Context, id string) (*position.Position, error)
	GetByAccount(ctx context.Context, accountID string) ([]*position.Position, error)
	Create(ctx context.Context, pos *position.Position) error
	Update(ctx context.Context, pos *position.Position) error
	Delete(ctx context.Context, id string) error
}

// OrderRepository defines order persistence operations
type OrderRepository interface {
	Get(ctx context.Context, id string) (*order.Order, error)
	GetPending(ctx context.Context, accountID string) ([]*order.Order, error)
	Create(ctx context.Context, ord *order.Order) error
	Update(ctx context.Context, ord *order.Order) error
	Cancel(ctx context.Context, id string) error
}

// TradeRepository defines trade persistence operations (read-only after creation)
type TradeRepository interface {
	Create(ctx context.Context, trade *trade.Trade) error
	GetByAccount(ctx context.Context, accountID string) ([]*trade.Trade, error)
	GetByPosition(ctx context.Context, positionID string) ([]*trade.Trade, error)
}
```

**Service interfaces:**
```go
// backend/internal/ports/services.go
package ports

import (
	"context"

	"github.com/trading-engine/internal/domain/order"
	"github.com/trading-engine/internal/domain/position"
)

// TradingService defines trading operations
type TradingService interface {
	ExecuteBuy(ctx context.Context, order *order.Order) (*position.Position, error)
	ExecuteSell(ctx context.Context, order *order.Order) (*position.Position, error)
	ClosePosition(ctx context.Context, positionID string) error
	CalculateMargin(ctx context.Context, accountID string) error
}

// MarketDataService defines market data operations
type MarketDataService interface {
	GetCurrentPrice(ctx context.Context, symbol string) (bid, ask float64, error)
	SubscribeToSymbol(ctx context.Context, symbol string) error
}
```

**Checkpoint:** Interfaces defined, compile successfully

// Verify interface implementation at compile time
var _ ports.AccountRepository = (*AccountRepository)(nil)

func NewAccountRepository(db *pgxpool.Pool) *AccountRepository {
	return &AccountRepository{db: db}
}

func (r *AccountRepository) Get(ctx context.Context, id string) (*account.Account, error) {
	var acc account.Account

	err := r.db.QueryRow(ctx, `
		SELECT id, balance, equity, leverage, margin, free_margin
		FROM accounts WHERE id = $1

	if err != nil {
		return nil, fmt.Errorf("failed to get account %s: %w", id, err)
	}

	return &acc, nil
}

### Task 4: Refactor engine.go into Trading Service

**What:** Split 913-line engine.go into focused service modules

**Why:** Single responsibility, testability, maintainability

**Details:**

**Create trading service:**
```go
// backend/internal/adapters/service/trading.go
package service

import (
	"context"
	"fmt"

	"github.com/trading-engine/internal/domain/order"
	"github.com/trading-engine/internal/domain/position"
	"github.com/trading-engine/internal/ports"
	"github.com/trading-engine/internal/shared/logger"
)

// TradingService implements ports.TradingService
type TradingService struct {
	accounts  ports.AccountRepository
	positions ports.PositionRepository
	orders    ports.OrderRepository
	trades    ports.TradeRepository
	marketData ports.MarketDataService
}

var _ ports.TradingService = (*TradingService)(nil)

func NewTradingService(
	accounts ports.AccountRepository,
	positions ports.PositionRepository,
	orders ports.OrderRepository,
	trades ports.TradeRepository,
	marketData ports.MarketDataService,
) *TradingService {
	return &TradingService{
		accounts:   accounts,
		positions:  positions,
		orders:     orders,
		trades:     trades,
		marketData: marketData,
	}
}

func (s *TradingService) ExecuteBuy(ctx context.Context, ord *order.Order) (*position.Position, error) {
	// Get account
	acc, err := s.accounts.Get(ctx, ord.AccountID)
	if err != nil {
		return nil, fmt.Errorf("failed to get account: %w", err)
	}

	// Get current price
	bid, ask, err := s.marketData.GetCurrentPrice(ctx, ord.Symbol)
	if err != nil {
		return nil, fmt.Errorf("failed to get price for %s: %w", ord.Symbol, err)
	}

	// Validate account can open position
	required := ord.Volume.Mul(decimal.MustParse(ask))
	if !acc.CanOpenPosition(required) {
		return nil, fmt.Errorf("insufficient funds: required %s, available %s",
			required, acc.Balance)
	}

	// Create position
	pos := &position.Position{
		ID:        generateID(),
		AccountID: ord.AccountID,
		Symbol:    ord.Symbol,
		Side:      "BUY",
		Volume:    ord.Volume,
		OpenPrice: decimal.MustParse(ask),
	}

	if err := s.positions.Create(ctx, pos); err != nil {
		return nil, fmt.Errorf("failed to create position: %w", err)
	}

	// Update account balance
	acc.UpdateBalance(required.Neg())
	if err := s.accounts.Update(ctx, acc); err != nil {
		return nil, fmt.Errorf("failed to update account: %w", err)
	}

	logger.Default.Info("buy order executed",
		"order_id", ord.ID,
		"position_id", pos.ID,
		"symbol", ord.Symbol,
		"volume", ord.Volume,
		"price", ask,
	)

	return pos, nil
}

// ... implement ExecuteSell, ClosePosition, CalculateMargin
```

**Split into multiple service files if needed:**
- `trading.go` - Order execution (ExecuteBuy, ExecuteSell, ClosePosition)
- `margin.go` - Margin calculations (CalculateMargin, CheckMarginCall)
- `stopout.go` - Liquidation logic (ExecuteStopOut)
- `validation.go` - Order validation

Each file should be <400 lines.

**Checkpoint:** Trading service compiles, implements port interface, all tests pass

### Task 6: Wire Dependencies in main.go

**What:** Update cmd/server/main.go to construct clean architecture dependencies

**Why:** Dependency injection at application entry point

**Details:**

```go
// backend/cmd/server/main.go
package main

import (
	"context"
	"log"
	"os"

	"github.com/gin-gonic/gin"
	"github.com/trading-engine/internal/adapters/http"
	"github.com/trading-engine/internal/adapters/postgres"
	"github.com/trading-engine/internal/adapters/service"
	"github.com/trading-engine/internal/shared/logger"
	"github.com/trading-engine/backend/database"
)

func main() {
	// Initialize logger
	logger.Init(slog.LevelInfo)
	logger.Default.Info("Trading Engine starting")

	// Initialize database
	db, err := database.NewPool(context.Background())
	if err != nil {
		log.Fatal("Failed to connect to database:", err)
	}
	defer db.Close()

	// Initialize repositories (adapters)
	accountRepo := postgres.NewAccountRepository(db)
	positionRepo := postgres.NewPositionRepository(db)
	orderRepo := postgres.NewOrderRepository(db)
	tradeRepo := postgres.NewTradeRepository(db)

	// Initialize market data service (adapter)
	marketData := service.NewMarketDataService()

	// Initialize trading service (application service)
	tradingService := service.NewTradingService(
		accountRepo,
		positionRepo,
		orderRepo,
		tradeRepo,
		marketData,
	)

	// Initialize HTTP handlers
	handlers := http.NewHandlers(
		tradingService,
		accountRepo,
		positionRepo,
		orderRepo,
	)

	// Setup Gin router
	r := gin.Default()
	http.SetupRoutes(r, handlers)

	// Start server
	port := os.Getenv("PORT")
	if port == "" {
		port = "8080"
	}

	logger.Default.Info("Server listening", "port", port)
	if err := r.Run(":" + port); err != nil {
		log.Fatal("Failed to start server:", err)
	}
}
```

**Checkpoint:** Application starts, dependencies wired correctly, all endpoints work

## Anti-Patterns to Avoid

- **Over-engineering:** Don't create 20 directories for 5 files - start simple, add structure as needed
- **Circular dependencies:** domain → ports (interfaces), adapters → ports (implement), adapters → domain (use entities)
- **Leaking infrastructure into domain:** No database imports in domain/, no HTTP imports in domain/
- **Creating interfaces for everything:** Only create interfaces at boundaries (ports), not for every struct
