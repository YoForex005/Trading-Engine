# Plan: Code Duplication Elimination

**Phase:** 16 - Code Organization & Best Practices
**Wave:** 3 (parallel with 16-05)
**Dependencies:** 16-01 (linting), 16-02 (logging), 16-04 (backend refactoring), 16-05 (frontend refactoring)
**Scope:** ~35% context budget

---

## Goal

Detect and eliminate code duplication across backend and frontend using jscpd, extracting shared logic into reusable functions/hooks and enforcing DRY principle via CI/CD checks.

---

## Must-Haves

### Truths (what must be TRUE when done)

1. Code duplication <5% across entire codebase (measured by jscpd)
2. Duplicate backend logic extracted into internal/shared/ packages
3. Duplicate frontend logic extracted into shared/utils/ or custom hooks
4. jscpd runs in CI/CD and fails builds if duplication exceeds threshold
5. Zero critical duplications (>50 lines) remain

### Artifacts (files that must exist)

```
.jscpd.json                              # jscpd configuration
.github/workflows/duplication-check.yml  # CI/CD duplication check
backend/internal/shared/utils/           # Shared backend utilities
clients/desktop/src/shared/utils/        # Shared frontend utilities
```

### Key Links (code that must connect correctly)

- jscpd analyzes both backend/ and clients/desktop/src/
- CI/CD fails if duplication >5% or any critical duplications found
- Shared utilities imported by multiple modules

## Tasks

### Task 1: Install and Configure jscpd

**What:** Set up jscpd for duplicate code detection

**Why:** Automated detection across 150+ languages including Go and TypeScript

**Details:**

**Install jscpd:**
```bash
# Install globally or as dev dependency
bun add -D jscpd
```

**Create configuration:**
```json
// .jscpd.json
{
  "threshold": 5,           // Fail if >5% duplication
  "reporters": ["console", "html", "json"],
  "ignore": [
    "**/node_modules/**",
    "**/dist/**",
    "**/build/**",
    "**/*.test.ts",
    "**/*.test.tsx",
    "**/*.test.go",
    "**/testdata/**",
    "**/vendor/**",
    "**/*.pb.go"            // Ignore generated protobuf files
  ],
  "format": ["go", "typescript", "tsx"],
  "mode": "strict",
  "minLines": 5,            // Minimum 5 lines to consider duplication
  "minTokens": 50,          // Minimum 50 tokens to consider duplication
  "output": ".jscpd-report"
}
```

**Run baseline scan:**
```bash
npx jscpd . --threshold 0
```

This will generate a report showing current duplication levels.

**Checkpoint:** jscpd runs successfully, generates report

### Task 2: Analyze and Prioritize Duplications

**What:** Review jscpd report and identify high-priority duplications to fix

**Why:** Fix most impactful duplications first (large blocks, critical paths)

**Details:**

**Review HTML report:**
```bash
open .jscpd-report/html/index.html
```

**Prioritize by:**
1. **Size:** >50 lines = critical (fix immediately)
2. **Location:** Business logic > utility code > tests
3. **Frequency:** Copied 5+ times > 2-3 times

**Common duplication patterns to look for:**

**Backend (Go):**
- Database query boilerplate (SELECT * FROM ... WHERE id = $1)
- Error handling patterns (if err != nil, return fmt.Errorf)
- Validation logic (check balance, check margin, check limits)
- Logging patterns (logger.Info with same fields)

**Frontend (TypeScript):**
- API fetch calls (fetch → json → setState)
- Form validation (check required fields, format validation)
- WebSocket connection handling (onopen, onmessage, onerror, onclose)
- State management patterns (useState + useEffect)

**Create duplication backlog:**
```markdown

### Task 3: Extract Shared Backend Utilities

**What:** Create reusable functions in internal/shared/ for duplicated Go code

**Why:** DRY principle, single source of truth

**Details:**

**Example 1: Query error handling**

Current duplication:
```go
// Found in account_repo.go, position_repo.go, order_repo.go, trade_repo.go
err := r.db.QueryRow(ctx, "SELECT * FROM accounts WHERE id = $1", id).Scan(&acc)
if err != nil {
	if errors.Is(err, pgx.ErrNoRows) {
		return nil, errors.NewNotFound("account", id)
	}
	return nil, fmt.Errorf("failed to get account %s: %w", id, err)
}
```

Extract to utility:
```go
// backend/internal/shared/database/errors.go
package database

import (
	"errors as stderrors"
	"fmt"

	"github.com/jackc/pgx/v5"
	"github.com/trading-engine/internal/shared/errors"
)

// HandleQueryError wraps database query errors with appropriate types
func HandleQueryError(err error, resource, id string) error {
	if err == nil {
		return nil
	}

	if stderrors.Is(err, pgx.ErrNoRows) {
		return errors.NewNotFound(resource, id)
	}

	return fmt.Errorf("failed to get %s %s: %w", resource, id, err)
}

// Usage in repositories
err := r.db.QueryRow(ctx, "SELECT * FROM accounts WHERE id = $1", id).Scan(&acc)
if err := database.HandleQueryError(err, "account", id); err != nil {
	return nil, err
}
```

**Example 2: Validation helpers**

Extract to utility:
```go
// backend/internal/shared/validation/decimal.go
package validation

import (
	"fmt"

	"github.com/trading-engine/internal/shared/decimal"
	"github.com/trading-engine/internal/shared/errors"
)

// ValidatePositive checks if decimal is > 0
func ValidatePositive(value decimal.Decimal, fieldName string) error {
	if value.LessThanOrEqual(decimal.Zero) {
		return errors.NewValidation(fieldName, "must be positive")
	}
	return nil
}

// ValidateRange checks if decimal is within min/max bounds
func ValidateRange(value, min, max decimal.Decimal, fieldName string) error {
	if value.LessThan(min) || value.GreaterThan(max) {
		return errors.NewValidation(fieldName,
			fmt.Sprintf("must be between %s and %s", min, max))
	}
	return nil
}

// Usage
if err := validation.ValidatePositive(order.Volume, "volume"); err != nil {
	return err
}

if err := validation.ValidateRange(order.Volume, minVol, maxVol, "volume"); err != nil {
	return err
}
```

**Example 3: HTTP error responses**

Current duplication:
```go

c.JSON(404, gin.H{"error": "not found"})

c.JSON(400, gin.H{"error": err.Error()})

Extract to utility:
```go
// backend/internal/adapters/http/response.go
package http

import (
	"github.com/gin-gonic/gin"
	"github.com/trading-engine/internal/shared/logger"
)

// ErrorResponse sends standardized error JSON
func ErrorResponse(c *gin.Context, status int, message string, err error) {
	if err != nil {
		logger.Default.Error("request failed",
			"path", c.Request.URL.Path,
			"status", status,
			"error", err,
		)
	}

	c.JSON(status, gin.H{
		"error": message,
		"status": status,
	})
}

// Usage in handlers
ErrorResponse(c, 404, "account not found", err)
ErrorResponse(c, 500, "internal server error", err)
ErrorResponse(c, 400, "invalid input", err)
```

### Task 4: Extract Shared Frontend Utilities

**What:** Create reusable functions/hooks in shared/ for duplicated React code

**Why:** DRY principle, consistent patterns

**Details:**

**Example 1: API client (fetch boilerplate)**

Current duplication:
```typescript
// Found in 10+ components
const res = await fetch(`/api/accounts/${id}`);
if (!res.ok) throw new Error('Failed to fetch account');
const data = await res.json();
```

Extract to API client:
```typescript
// clients/desktop/src/shared/services/api.ts
export class ApiError extends Error {
  constructor(public status: number, message: string) {
    super(message);
  }
}

export const api = {
  async get<T>(endpoint: string): Promise<T> {
    const res = await fetch(endpoint);
    if (!res.ok) {
      throw new ApiError(res.status, `GET ${endpoint} failed: ${res.statusText}`);
    }
    return res.json();
  },

  async post<T>(endpoint: string, data: unknown): Promise<T> {
    const res = await fetch(endpoint, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data),
    });
    if (!res.ok) {
      throw new ApiError(res.status, `POST ${endpoint} failed: ${res.statusText}`);
    }
    return res.json();
  },

  async delete(endpoint: string): Promise<void> {
    const res = await fetch(endpoint, { method: 'DELETE' });
    if (!res.ok) {
      throw new ApiError(res.status, `DELETE ${endpoint} failed: ${res.statusText}`);
    }
  },
};

// Usage
const account = await api.get<Account>(`/api/accounts/${id}`);
await api.post('/api/orders', { symbol: 'EURUSD', volume: 1.0 });
await api.delete(`/api/positions/${posId}`);
```

// clients/desktop/src/shared/utils/validation.ts
export const validators = {
  required(value: string | number | null | undefined, fieldName: string): string | null {
    if (value === null || value === undefined || value === '') {
      return `${fieldName} is required`;
    }
    return null;
  },

  positive(value: number, fieldName: string): string | null {
    if (value <= 0) {
      return `${fieldName} must be positive`;
    }
    return null;
  },

  range(value: number, min: number, max: number, fieldName: string): string | null {
    if (value < min || value > max) {
      return `${fieldName} must be between ${min} and ${max}`;
    }
    return null;
  },
};

// Usage
const symbolError = validators.required(symbol, 'Symbol');
if (symbolError) {
  setError(symbolError);
  return;
}

const volumeError = validators.positive(volume, 'Volume');
if (volumeError) {
  setError(volumeError);
  return;
}
```

### Task 5: Add jscpd to CI/CD Pipeline

**What:** Run jscpd in GitHub Actions to prevent new duplications

**Why:** Enforce code quality standards automatically

**Details:**

```yaml
# .github/workflows/duplication-check.yml
name: Code Duplication Check

on:
  pull_request:
  push:
    branches: [main]

jobs:
  check-duplication:
    name: Check Code Duplication
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: oven-sh/setup-bun@v1

      - name: Install jscpd
        run: bun add -g jscpd

      - name: Run duplication check
        run: jscpd . --threshold 5 --exitCode 1

      - name: Upload duplication report
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: jscpd-report
          path: .jscpd-report/
          retention-days: 7
```

**Local pre-commit hook (optional):**
```bash
# .git/hooks/pre-commit
#!/bin/bash
echo "Checking for code duplication..."
npx jscpd . --threshold 5 --exitCode 1

if [ $? -ne 0 ]; then
  echo "❌ Code duplication exceeds threshold. Fix duplications before committing."
  echo "   View report: open .jscpd-report/html/index.html"
  exit 1
fi

echo "✓ Duplication check passed"
```

**Checkpoint:** CI/CD fails on excessive duplication

### Task 6: Document DRY Best Practices

**What:** Add guidelines for avoiding code duplication

**Why:** Prevent future duplications

**Details:**

Add to CONTRIBUTING.md:
```markdown
## Code Duplication (DRY Principle)

### Avoid Copy-Paste Programming

Before copying code, ask:
1. Can this be extracted into a shared function/hook?
2. Is this logic truly identical, or just similar?
3. Will this logic change together across all copies?

If yes to all three, extract it.

### Extracting Shared Logic

**Backend (Go):**
- Shared utilities → `backend/internal/shared/`
- Database helpers → `backend/internal/shared/database/`
- Validation → `backend/internal/shared/validation/`

**Frontend (TypeScript):**
- Shared hooks → `clients/desktop/src/shared/hooks/`
- Shared components → `clients/desktop/src/shared/components/`
- Utilities → `clients/desktop/src/shared/utils/`
- API client → `clients/desktop/src/shared/services/`

### CI/CD Enforcement

Code duplication is checked automatically via jscpd:
- **Threshold:** 5% (builds fail if exceeded)
- **Report:** Available in GitHub Actions artifacts

View duplication report locally:
```bash
npx jscpd .
open .jscpd-report/html/index.html
```

### When Duplication is Acceptable

- Test fixtures (similar setup, different assertions)
- Configuration files (similar structure, different values)
- Small snippets (<5 lines) where abstraction adds complexity
```

## Anti-Patterns to Avoid

- **Over-abstraction:** Don't create a utility for 3 lines of code used twice
- **Premature extraction:** Wait until code is duplicated 3+ times before extracting
- **Ignoring context:** Similar code with different purposes shouldn't be merged
- **Breaking semantics:** Don't force-fit unrelated code into shared utility just to reduce duplication percentage

**Rule of Three:** Extract when duplicated 3+ times, not at first duplication.
