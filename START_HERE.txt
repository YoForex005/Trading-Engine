WHAT GETS MONITORED
===================

PROCESS METRICS (go.exe/server.exe)
  - CPU Usage (%)              How much processor the server uses
  - Memory Usage (MB)          RAM consumed by server
  - Private Memory (MB)        Memory exclusively for server
  - Thread Count               Number of active threads

SYSTEM METRICS
  - System CPU (%)             Total computer CPU usage
  - System Memory (MB)         Total computer memory in use
  - Memory Utilization (%)     Percentage of available RAM used
  - Available Memory (MB)      Free RAM still available

STABILITY CHECKS
  - Memory Leaks               Does memory keep growing?
  - CPU Spikes                 Are there sudden CPU peaks?
  - Thread Stability           Do threads keep increasing?
  - System Responsiveness      Does system slow down?

KEY METRICS EXPLAINED
=====================

Peak CPU Usage (%)
  What: Maximum CPU at any point during test
  Good: < 50% (ideal), < 80% (acceptable)
  Bad:  > 90% (critical)

Average CPU Usage (%)
  What: Mean CPU usage throughout test
  Good: 10-30% (typical streaming)
  Bad:  > 60% (continuously high)

Peak Memory (MB)
  What: Maximum RAM used at any point
  Good: < 500MB (ideal), < 1500MB (acceptable)
  Bad:  > 2000MB (critical)

Average Memory (MB)
  What: Mean RAM usage throughout test
  Good: 100-300MB (typical)
  Bad:  > 1000MB (excessive)

Memory Growth (MB)
  What: How much memory increased from start to end
  Good: < 50MB (minimal growth)
  Bad:  > 100MB (possible leak)

Memory Growth Rate (MB/minute)
  What: How fast memory is increasing per minute
  Good: < 10 MB/min (acceptable)
  Bad:  > 50 MB/min (probable leak)

COMMON ISSUES & QUICK FIXES
=============================

Issue: "Cannot reach server at localhost:8080"
Fix:   - Check server.exe exists at backend\cmd\server\
       - Verify port 8080 is not in use
       - Check Windows Firewall isn't blocking

Issue: "High memory usage detected"
Fix:   - This might be normal for streaming
       - Check if memory stabilizes (plateaus)
       - If memory keeps growing, investigate for leak

Issue: "High CPU usage"
Fix:   - Normal under load
       - If sustained > 80%, code may need optimization
       - Try lower request rates in load test

Issue: "Script won't run"
Fix:   - Run: powershell -ExecutionPolicy Bypass
       - Verify all files in same directory
       - Check server.exe exists
